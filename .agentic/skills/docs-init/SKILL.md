# docs-init

Genera una documentación mínima para este repositorio.

Primero lee:

docs/documentation-methodology.md

Después inspecciona el código del repositorio y crea, si no existen:

docs/README.md
docs/architecture.md
docs/api.md
docs/operations.md
docs/notes/

Reglas:

- Documenta solo lo que no sea evidente leyendo el código.
- Mantén los documentos cortos.
- No inventes APIs, servicios, comandos ni arquitectura.
- Si algo no está claro, márcalo como pendiente de verificar.
- No borres documentación existente.
- Si un documento ya existe, propón cambios antes de sobrescribirlo.

Resultado esperado:

- crear o proponer la documentación mínima inicial;
- explicar brevemente qué documentos se han creado o qué queda pendiente.
